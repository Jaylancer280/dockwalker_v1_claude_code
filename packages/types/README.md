# @dockwalker/types

Shared TypeScript type definitions for the DockWalker monorepo. This package has no runtime dependencies — it exports only types.

## Files

| File            | Contents                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| --------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `src/enums.ts`  | `RoleContext`, `IdentityType`, `ApplicationStatus` (includes `shortlisted`), `VesselType`, `MealOption`                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| `src/events.ts` | `EventType` (all domain event names including `ENGAGEMENT.POSTPONEMENT_*`, `ENGAGEMENT.WORK_STARTED`, `ENGAGEMENT.WORK_STARTED_CONFIRMED`, `DAYWORK.RELISTED`, `ENGAGEMENT.CANCELLATION_RATED_BY_*`, `CHECKLIST.SET`, `CHECKLIST.ITEM_TOGGLED`), `AggregateType` (includes `engagement`, `checklist`), `DomainEvent` interface, `EventPayloadMap` (typed payload per event for compile-time validation). `VESSEL.CREATED` carries `loa_meters: number` (exact LOA); `VESSEL.UPDATED` carries optional `loa_meters`. `CHECKLIST.SET` carries `engagement_id` and `items` array (`{ id, label, value }`). `CHECKLIST.ITEM_TOGGLED` carries `engagement_id`, `item_id`, and `checked` boolean. `AVAILABILITY.SET` carries `start_date`, `end_date`, `expires_at`, `city_id` (nullable string — FK to cities table for town-level location), optional `port_id` (nullable string — FK to ports table for optional port-level granularity), and optional `not_available` boolean (explicit "not available" declaration). |
| `src/models.ts` | Entity interfaces: `Person`, `CrewProfile`, `AgentProfile`, `Vessel`, `Daywork`, `Application`, `AvailabilityWindow`, `Message`, `Engagement`, `EngagementRating`                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| `src/index.ts`  | Barrel re-export of all the above                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |

## Usage

```ts
import type { DomainEvent, EventType, EventPayloadMap, RoleContext } from '@dockwalker/types';
```

## Scripts

```bash
npm run type-check   # tsc --noEmit
```
